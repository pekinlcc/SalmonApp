IconRail 图标方案

每列是同一组 8 个图标（首页 / 联系人 / 邮件 / 日历 / 待办 / Topic / 搜索 / 设置）。第二项「联系人」设为 active，邮件带 12 角标，CLI 状态点保留。1:1 还原真实 rail。

方案 A · 线框 → 实心

默认线条 (ink-700)，选中时切换为实心 + salmon-700。iOS / Notion 常用，层级最清晰。

方案 B · 纯线条

所有状态都是线条，只用颜色 (ink-500 → salmon-700) + 极轻微的描边加粗区分。最克制、最「极简」。

方案 C · 全实心

默认 ink-700 + 55% 透明度，选中切换为 salmon-700 实心。视觉重量最足，跟「饱和的桌面应用」气质一致。

方案 D · 中性墨色

图标始终中性 (ink-500 → ink-900)，不用 salmon；状态完全靠 pill + 左竖条传达。最克制，跟未来其它颜色需求最不冲突。

说明
  • 四列都使用同一套 SVG 几何（24×24，1.6px 线宽），区别只在填充策略和颜色策略。
  • 都保留了原来的 active 视觉：salmon-100 pill + salmon-700 左侧 3px 指示条 + 右上角 badge。
  • 没有任何系统 emoji，跨平台显示一致。
  • A / C 的「线 vs 实」对比更强；B 最简；D 把颜色全部交给容器（pill 和指示条），图标本身只承担形状识别。
