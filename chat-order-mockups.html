权衡对比

A 内联交错B 折叠思考C 段落 chips
时间顺序保真 ✓ 完全按 stream 到达顺序 ✗ 全部塞进折叠块,顺序仍在但被压扁 ~ 段内打散成 chips,跨段顺序保留
第一眼看清答案 ~ 答案在最末,要滚到底 ✓ 答案永远在最显眼位置 ✓ 文字段落主导,chip 只作旁注
调试 / 复盘工具调用 ✓ 直接平铺,所有参数一眼可见 ~ 默认收起,要点开 ✗ chip 太精简,要点开右栏看详情
大量工具调用时(20+) ✗ 中栏被工具卡片塞爆,文字被淹没 ✓ 折叠后视觉负担恒定 ✓ chip 是最紧凑的
实现复杂度 低 — 只要改数据结构成 ordered blocks 数组 中 — 需要跟踪一组 tool 的开始/结束 + 总时长 高 — 需要把 tool 按"产出下一段文字之前"的窗口分组
跟现有 Tool Card 渲染兼容 ✓ 直接复用 ✓ 复用 + 包一层 details ✗ 要写新的 chip 组件
我的建议: A(内联交错)。 Salmon 的目标用户是真在用 CLI 的开发者,他们要看的就是"AI 是怎么一步步推进的",B 把这个隐藏掉就违背了产品定位;C 看着干净但代价是去找右栏才能看 tool 详情,多一次点击。 唯一的 A 痛点是工具调用很多时候中栏会很长,但这可以通过单个 ToolCard 默认折叠 + Topic 内"折叠所有工具"快捷键解决,不需要换整个范式。
