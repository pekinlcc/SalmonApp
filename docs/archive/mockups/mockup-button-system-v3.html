SalmonApp 全局组件规范 v1.19.2 — 按钮 + 表单控件

v1.19.0 把按钮统一了，但 <select> 和 <input> 还在裸用浏览器默认样式 — Tasks 页那个"全部账号 ⇕"下拉、SettingsDialog 的输入框、Compose 的发件人 select……夹在精心调过的按钮中间看起来像 2005 年的 Windows 应用。这一版同时收口两件事：

  1. 按钮系统：.btn-ghost 加可见边框；新增 .btn-quiet 承接"真要无边框"的场景。
  2. 表单控件系统：新增 .input / .select / .textarea，跟 .btn 同 DNA（同高度、同圆角、同主题色 focus），原生 <select> 的丑陋 ⇕ 替换成自绘 SVG chevron。
核心改动：
  1. .btn-ghost 改造：从「透明边框 + 灰文本」改为「白底 + salmon-300 边框 + salmon-700 文字 + 600 字重」——和 Briefing 刷新按钮完全一致。
  2. 新增 .btn-quiet：把"真的不想要边框"的场景（模态框 × 关闭、RightPane 折叠箭头、单纯的图标 toolbar 等）单独承接。
  3. 合并冗余：删除 .brief-refresh-btn 和 .welcome-refresh-btn 两个一次性样式，统一用 .btn .btn-ghost + 旋转 spinner 子元素。

5. 决策树（什么时候用哪个变体）

按钮

问题选择
当前界面的"主推荐动作"？是 → .btn-primary
破坏性（删除 / 拒绝 / 不重要）？是 → .btn-danger（重要时叠加 .btn-primary 变实心红）
header / 侧栏 / 工具区的强调辅助？是 → .btn-ghost（带 salmon 边框）
模态框 Cancel、低优先级附属？是 → 默认 .btn（中性）
模态框 ×、折叠箭头、单纯图标？是 → .btn-quiet（无边框）

表单控件

输入类型选择
单行文本.input + 可选 .input-sm / .input-lg

多行 / 备注 / 邮件正文.textarea
勾选框 / 单选框.check + .check-row
搜索框（带图标）.search-input 容器 + .input
验证错误加 .is-error 修饰器
