4实施成本（实话）

v0.10 工程量分布（估算）

  1. Roost 阶段（~半天）：Rust，按联系人聚合本地 mail_messages + calendar_events。已经有 contacts 表打底，加一个 ContactBundle 结构。
  2. ContactPulse agent（~1 天）：复用现有 engine.rs 的 spawn 逻辑，每联系人一次 claude --print 子进程；JSON 输出 schema（items[] + suggestedActions + steps）。串行调用，进度事件 emit 给 FE。
  3. MailBriefing agent（~半天）：第二次 LLM 把所有联系人的 items 合并。
  4. Cross-link agent（~半天）：单次轻量调用，喂两侧输出 + 关键字提取（联系人邮箱 vs Topic 标题/workdir/最近消息）。
  5. Rubric 系统（~1 天）：~/.config/salmonapp/rubric.md 加载 + feedback.log append + Editor agent（10 条或 24h 触发）。
  6. Writer / Calendar Extractor / Task Extractor agents（~1 天）：用户点 step 时按需 spawn。
  7. FE 改写（~1.5 天）：home feed 改成 Cross-link 优先 + 单卡 suggestedActions + 内联回复审稿区 + step ⚙ 调整面板 + ack toast。
  8. Schema + DB（~半天）：briefings 表加 contact_bundles JSON + cross_links JSON，alpha schema 已留好不用迁。
  9. 增量重算 + debounce（~半天）：新邮件到 → 受影响联系人 dirty → 30 秒后 trigger pipeline。

总计 ~ 6 天实打实的工程（v0.10 alpha.1 → alpha.5），中间需要你给 OAuth 测过 / 给我反馈 prompt 写得对不对 / rubric 看着合不合理。 不是一晚上能搞完的。
