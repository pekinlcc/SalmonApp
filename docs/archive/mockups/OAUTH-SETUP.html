✦ SalmonApp OAuth 配置

把 Gmail / Outlook 接进 SalmonApp 一共三件事：① 注册 Google ② 注册 Microsoft ③ 凭据填进 oauth_config.toml。每个 ~15 分钟。

TL;DR — 我最终要从你那里拿到这三个值
Google client_id: ……..apps.googleusercontent.com
Google client_secret: GOCSPX-………
Microsoft client_id: ……-……-……-……-……

Microsoft 不需要 secret（桌面公共客户端走 PKCE）。只想接 Gmail 就只给前两个；只想接 Outlook 就只给最后一个。

步骤

  1. 准备：建配置文件
  2. 注册 Google（Gmail + Calendar + Contacts）
  3. 注册 Microsoft（Outlook + Graph）
  4. 把凭据交给我 → 我填进 toml → 你重启 SalmonApp

0准备：建配置文件

在终端里跑这一行，把模板复制成真正会被加载的文件（已 .gitignore，不会被 commit）：

cp "/home/bytedance/桌面/Salmon App/salmon/src-tauri/oauth_config.toml.example" \
   "/home/bytedance/桌面/Salmon App/salmon/src-tauri/oauth_config.toml"

1Google · Gmail + Calendar + Contacts

需要 Google 账号。个人 gmail.com 就行。

1
打开 console.cloud.google.com，顶部下拉点 新建项目，名字随便取（例：salmonapp-local），点 Create。
2
左边菜单 APIs & Services → Library，依次搜索并 Enable 这三个：
  • Gmail API
  • Google Calendar API
  • People API（联系人用）
3
APIs & Services → OAuth consent screen：
  • User Type 选 External（个人账号也能用）
  • App name: SalmonApp
  • User support email / Developer contact: 填你自己的邮箱
  • 下一步 Scopes 那一页直接 Save and continue，不用勾
  • Test users 里 Add Users，把你自己的 Gmail 加进去（不发布给公众，只给你自己测）
4
APIs & Services → Credentials → + Create Credentials → OAuth client ID：
  • Application type: Desktop app（一定要选 Desktop，不是 Web）
  • Name 随便
  • 点 Create
5
弹出的对话框里有 Client ID 和 Client Secret，复制下来给我。后面随时能在 Credentials 列表点进去看。
这一步结束，你会有：
Google client_id: 类似 123456789012-abc...xyz.apps.googleusercontent.com
Google client_secret: 类似 GOCSPX-AbCdEfGhIjKlMnOpQrStUv

2Microsoft · Outlook + Graph

需要 Microsoft 账号（个人 outlook.com / 工作 Office365 都行）。

1
打开 portal.azure.com，顶部搜索栏输入 App registrations，回车进去。
2
点 + New registration：
  • Name: SalmonApp
  • Supported account types：选 Accounts in any organizational directory and personal Microsoft accounts（第三项；这样个人 outlook.com 也能登）
  • Redirect URI：类型选 Public client/native (mobile & desktop)，URI 填 http://localhost（Azure 会接受这个并放行动态端口）
  • 点 Register
3
新建好的页面顶部就显示 Application (client) ID，复制下来给我，这就是 Microsoft 的 client_id。
4
左边 Authentication → 拉到底 Advanced settings → 找到 Allow public client flows，改成 Yes，点最下面 Save。
⚠️ 必须打开
没开这个，PKCE 流会被 Azure 拒掉，登录到一半失败。
5
左边 API permissions → + Add a permission → Microsoft Graph → Delegated permissions，勾上：
  • Mail.ReadWrite
  • Mail.Send
  • Calendars.ReadWrite
  • Contacts.Read
  • User.Read（默认就在）
  • offline_access
点 Add permissions。个人账号下，"Grant admin consent" 不用点；工作账号需要管理员授权。
这一步结束，你会有：
Microsoft client_id: 类似 12345678-abcd-1234-abcd-1234567890ab

3把凭据交给我 → 我填 toml → 你重启

1
把上面三个值（或只填你需要的那 provider）直接贴在 SalmonApp 聊天里给我，格式举例： 
Google client_id: 123456-xxxx.apps.googleusercontent.com
Google client_secret: GOCSPX-xxxxxxxxxxxxxxxx
Microsoft client_id: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
2
我把这些值写进 salmon/src-tauri/oauth_config.toml。这个文件是 gitignored，不会进任何 commit。
3
你 关掉 SalmonApp 再重新打开（backend 只在启动时读 toml）。
4
点左侧栏 📧 邮件 → ＋ 添加 Gmail / Outlook，系统浏览器会自动打开 OAuth 授权页，登录后 5 分钟内回到 SalmonApp 就好。
几个小提醒
  • 同一组凭据可以重复用，加多个 Gmail / Outlook 账号都行。
  • 只配 Google 就行（不想用 Outlook 的话），把 [microsoft] 留空即可。
  • token 存在本地 SQLite（~/.local/share/app.salmonapp.desktop/salmon.db），不会上传到任何服务器。
  • OAuth 配置改了之后一定要重启 SalmonApp 才生效。
